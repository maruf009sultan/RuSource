## ✨ What is RuSource?

**RuSource** *(Russian + Source)* is a free, open-source, curated directory of **749+ hand-picked resources for learning the Russian language**. It's the interactive web companion to the [awesome-russian-language](https://github.com/maruf009sultan/awesome-russian-language) dataset, transformed into a blazing-fast, searchable, filterable, SEO-optimized Progressive Web App.

Instead of scrolling through an endless GitHub markdown table, RuSource gives you:

- 🔍 **Fuzzy search** across every resource — find anything in milliseconds
- 🏷️ **32 curated categories** — from Alphabet & Phonetics to GitHub Repos & NLP
- 📊 **CEFR level filtering** — A1 (beginner) through C2 (mastery)
- 💰 **Pricing filters** — find free resources instantly
- 📱 **Install as a PWA** — works offline, like a native app
- ⭐ **Favorites & history** — saved locally, no account needed
- 🗺️ **A1→C2 roadmap** — structured learning path with milestones
- 🎲 **Resource of the day** — discover something new every 24 hours

**No sign-up. No paywall. No tracking. No ads.** Just pure, structured knowledge.

## 🗺️ The A1 → C2 Learning Roadmap

RuSource includes a **comprehensive 6-stage roadmap** from complete beginner (A1) to near-native mastery (C2), based on the Common European Framework of Reference for Languages (CEFR):

| Stage | Level | Hours | What You'll Achieve |
|-------|-------|-------|-------------------|
| 🟢 | **A1** — Beginner | 0–100 | Read Cyrillic, introduce yourself, order food, basic directions |
| 🟡 | **A2** — Elementary | 100–200 | Simple conversations, past/future tense, shopping, transport |
| 🟠 | **B1** — Intermediate | 200–400 | Travel independently, express opinions, read simple news |
| 🔵 | **B2** — Upper Intermediate | 400–600 | Watch movies without subtitles, write essays, debate topics |
| 🟣 | **C1** — Advanced | 600–800 | Professional communication, understand sarcasm, read literature |
| 🔴 | **C2** — Proficiency | 800–1200 | Near-native fluency, academic writing, cultural nuance mastery |

Each stage comes with **daily routines**, **grammar focus areas**, **common pitfalls**, and **milestone checkpoints**. Visit [/roadmap](https://rusource.org/roadmap) to see the full guide.

### Architecture Highlights

- **Zero external API calls** — All 749 resources are bundled as static JSON. No backend, no database, no auth, no API keys
- **Edge SSR** — Server-rendered HTML at the Cloudflare edge for every page, then hydrated client-side
- **Manual chunk splitting** — `fuse.js` → search chunk, `lucide` → icons chunk, `sonner` → toast chunk
- **CSS-only animations** — IntersectionObserver + CSS classes. Zero JavaScript animation libraries at runtime
- **Content-visibility** — Below-fold sections use `content-visibility: auto` for faster initial paint
- **Font preloading** — Critical fonts preload + font-display: swap for zero FOIT

## 🎨 Design System — "Editorial Constructivist"

RuSource's visual identity is inspired by **Russian Constructivism** — the early 20th-century art movement defined by bold geometry, stark contrasts, and purpose-driven design. Every pixel serves the content.

### Color System (oklch)

| Token | Light Mode | Dark Mode | Role |
|-------|-----------|-----------|------|
| **Cream** | `oklch(0.962 0.018 85)` | — | Warm off-white background |
| **Ink** | `oklch(0.16 0.012 270)` | — | Deep near-black foreground |
| **Signal** 🇷🇺 | `oklch(0.605 0.215 27)` | `oklch(0.66 0.22 27)` | **Russian red** — primary accent |
| **Gold** | `oklch(0.78 0.14 80)` | — | Accent for freemium badges |

### Typography

| Font | Weight | Purpose |
|------|--------|---------|
| **Unbounded** | 800–900 | Display headings — geometric, bold, unmistakable |
| **Inter** | 400–700 | Body text — optimized for screens, highly legible |
| **JetBrains Mono** | 400–700 | Labels, badges, code — monospace precision |

### Signature Design Elements

- **Brutal shadows**: `6px 6px 0 0 var(--color-ink)` (light) / `var(--color-signal)` (dark) — no soft drops, no blurs
- **Hard edges**: Minimal border-radius (0.25rem max) — no rounded corners culture
- **Monospace labels**: `tracking-widest uppercase` everywhere — section markers like `§ 01 / Categories`
- **Cyrillic accents**: "РУССКИЙ ЯЗЫК" hero text with animated shimmer, "Не найдено" (404), "Ошибка" (500)
- **Starfield background**: 3-layer CSS-only animated star field with drift + twinkle
- **Marquee**: Scrolling Russian phrases — Привет, Спасибо, Пожалуйста, Добро пожаловать
- **Aurora blob**: Floating gradient accent for visual depth
- **Grid overlay**: Subtle background grid pattern for structure

## 📱 PWA — Install Like a Native App

RuSource is a **fully installable Progressive Web App**. On supported browsers (Chrome, Edge, Samsung Internet), users get a native-like install experience:

### How Installation Works

1. **Browser detects installability** — Chrome checks all PWA criteria (manifest, service worker, HTTPS, icons)
2. **Custom install banner appears** — A floating "Install RuSource" banner slides in from the bottom
3. **User taps "Install"** — The browser's native install dialog opens
4. **App installs to home screen** — Icon added to home screen, launches in standalone mode (no browser chrome)
5. **Works offline** — Service worker caches fonts, CSS, JS, and HTML for offline access

### Service Worker Caching Strategy

| Resource Type | Strategy | Why |
|--------------|----------|-----|
| Fonts (woff2) | **Cache-first** | Fonts never change — cache for 1 year |
| CSS & JS chunks | **Stale-while-revalidate** | Serve from cache, update in background |
| HTML pages | **Network-first** | Always try fresh content, fallback to cache |
| `sw.js` itself | **Network-only** | Always fetch latest service worker from server |

### PWA Requirements Checklist

- ✅ Web app manifest with `name`, `short_name`, `icons`, `start_url`, `display: standalone`
- ✅ 192×192 and 512×512 PNG icons (purpose: any)
- ✅ 512×512 maskable icon (safe zone: 80%)
- ✅ Service worker registered on every page
- ✅ Served over HTTPS (Cloudflare)
- ✅ `theme_color` matches meta tag (`#1a1620`)
- ✅ Custom install prompt (suppresses Chrome's mini-infobar)
- ✅ Offline fallback page

## ⚡ Performance

RuSource is built for speed. Every millisecond counts.

| Metric | Target | How |
|--------|--------|-----|
| **TTFB** | < 50ms | Edge SSR via Cloudflare Workers in 300+ cities |
| **FCP** | < 1.0s | Self-hosted fonts, font preloading, CSS-only animations |
| **LCP** | < 1.5s | No render-blocking resources, content-visibility |
| **CLS** | < 0.05 | Explicit widths/heights, font-display: swap |
| **TTI** | < 2.0s | Manual chunk splitting, lazy routes |
| **Bundle** | Minimal | Fuse.js, Lucide, and Sonner in separate async chunks |

### Performance Optimizations

- **Self-hosted fonts** — No Google Fonts `@import` blocking render
- **Font preloading** — Critical fonts (`<link rel="preload">`) for instant text display
- **CSS-only animations** — IntersectionObserver + CSS classes, zero JS animation runtime
- **Content-visibility: auto** — Below-fold sections skip rendering until needed
- **Manual chunk splitting** — Search (Fuse.js), icons (Lucide), toasts (Sonner) loaded async
- **Aggressive caching** — Fonts: 1yr immutable, CSS/JS: immutable, HTML: must-revalidate
- **Image-free design** — No raster images except PWA icons; everything is CSS/SVG/fonts
- **Tree-shaking** — Side-effect-free package, Lucide icons imported individually

## 🔒 Security

RuSource implements defense-in-depth security headers via Cloudflare's `_headers` file:

```
Content-Security-Policy: default-src 'self'; script-src 'self' 'sha256-mXMvbGg8IGPBUDuQD6F2K3zb+/IAQqaXsop+/UmZk9I=' https://www.googletagmanager.com https://www.gstatic.com; ...
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

- **No `unsafe-inline` for scripts** — GTM inline snippet is hash-allowed via `sha256-...` in CSP
- **No `unsafe-eval`** — Strict CSP, no eval allowed anywhere
- **X-Frame-Options: DENY** — Cannot be embedded in iframes (clickjacking protection)
- **frame-ancestors: 'none'** — CSP-level iframe blocking as backup
- **Service worker never cached** — `sw.js` served with `Cache-Control: no-cache` to ensure updates propagate immediately

## 🚀 Getting Started

### Prerequisites

- **Node.js** 20+ or **Bun** 1.0+
- A **Cloudflare account** (for deployment)

### Local Development

```bash
# Clone the repository
git clone https://github.com/maruf009sultan/RuSource.git
cd RuSource

# Install dependencies
npm install

# Start dev server
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) — changes hot-reload instantly.

### Build for Production

```bash
npm run build
```

Output goes to `.output/` — ready for Cloudflare Workers deployment.

### Preview Production Build Locally

```bash
npm run preview
```

## ⌨️ Keyboard Shortcuts

RuSource supports power-user keyboard shortcuts for fast navigation:

| Key | Action |
|-----|--------|
| `/` or `s` | Jump to search (Browse page) |
| `r` | Go to Resource of the Day |
| `f` | Go to Favorites |
| `Escape` | Unfocus / close dialogs |

## 🤝 Contributing

Contributions are welcome! Here's how you can help:

### Add a Resource

1. Fork the [upstream dataset](https://github.com/maruf009sultan/awesome-russian-language)
2. Add your resource to the appropriate category in the dataset
3. Submit a pull request

### Report an Issue

Found a broken link, incorrect level, or missing category? [Open an issue](https://github.com/maruf009sultan/RuSource/issues).

### Improve the Site

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request
